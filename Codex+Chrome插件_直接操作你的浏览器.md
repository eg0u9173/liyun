# 它具体能做什么？ 

OpenAI官方提到了几个典型的场景：

## **A.数据提取和报表生成：**

你每天要登录某个内部后台，把昨天的销售数据复制到Excel里做汇总，这套操作需要40分钟。现在你可以告诉Codex："每天早上9点，帮我把后台昨天的销售数据整理成报表，发到团队邮箱。"它会自动登录、提取数据、生成表格、发送邮件，你第二天早上只需要打开邮箱看结论。



## **B.CRM批量更新：**

销售团队每天都花大量时间在CRM系统里录数据、改状态、查客户。以前这些事情需要人坐在电脑前一个个操作，现在Codex可以直接帮你完成。你给它一份客户名单，让它帮你批量更新，它会自己登录、自己操作、自己验证结果。



## **C.表单批量填写：**

你要注册一批账号，或者要在一堆平台里录入同样的信息，只需要把数据给Codex，告诉它"帮我把这些填进去"，它会按照你的要求一个个完成。

这些事情有一个共同的特点：它们以前都需要你亲自动手，现在你只需要说清楚要什么。

# 第二步：**怎么在新对话里调用Chrome？**

安装好之后，不是说它会自动每次都接管我们的浏览器。我们需要在新对话里手动引用它。



## **方式一（推荐）：@Chrome + 任务描述**

直接在输入框打 `@Chrome`，后面跟你要做的事。比如：

`@Chrome 帮我登录 Salesforce ，根据这些笔记更新客户记录`

`@Chrome 打开我的 GitHub ，找到最近三个 PR ，把每个 PR 的改动要点汇总成表格`

Codex 会自动识别你要用浏览器，然后规划执行步骤。



## **方式二：/Chrome 命令**

输入 `/Chrome`，从弹出的插件列表里选 Chrome ，然后描述任务。跟方式一本质上一样，多了一步选择。



## **方式三：裸描述**

不加任何前缀，直接描述浏览器相关任务。比如"帮我在飞书文档里找上周的会议纪要"。 Codex 碰到浏览器操作类任务会自动建议使用 Chrome 插件。



这样就能把Chrome插件引用进当前任务里。



这个交互设计我觉得还挺合理的，因为它不是默认强行接管你的浏览器，而是你明确告诉它，欸，我这次的任务，需要你用Chrome来做。



它不是偷偷摸摸在后台乱跑，而是你给它一个任务，它才开始行动。

# **它适合做哪些事情？**



我觉得这个Chrome插件，短期内最适合的不是让AI完全替我们工作。

而是让它帮你处理那些重复、琐碎、路径明确、又离不开浏览器操作的事。

比如：

- 自动打开指定网站，登录后进入某个后台
- 在 B 站、YouTube、小红书上搜索特定类型的教程或案例
- 整理竞品网站信息
- 帮你打开 Gemini、即梦、可灵，直接开始生成素材
- 查找某个工具的官方文档
- 进入网页后台检查数据
- 在多个标签页之间快速切换资料
- 根据网页内容做初步总结
- 完成一些固定流程的网页操作

# Codex Chrome插件**和内置浏览器有什么区别**



Codex 内置浏览器：适合本地开发预览、公开页面、无需登录的网页。

局限：不支持您的常规浏览器资料、Cookie、扩展程序和已登录页面。

结论：当需要登录状态或浏览器扩展能力时，请使用常规浏览器或 Codex Chrome 扩展。

![图片展示了Codex应用内浏览器的相关信息。其可在Codex上预览和评论Web应用程序，能与Codex在同一线程中共享已渲染网页视图，适用于本地开发服务器、基于文件的预览及无需登录的公共页面。可从工具栏、单击URL等方式打开，快捷键为Mac的Cmd + Shift + B和Windows的Ctrl + Shift + B 。但它不支持身份验证流程等，仅适用于Codex无需登录即可打开的页面。与上下文关系为：上下文提及不同场景下浏览器使用选择，此图介绍了Codex应用内浏览器的功能与适用范围。](https://feishu.cn/file/YD9xbiPNHo6c4Tx7l7kcd1donye)



- 如果只是看一个公开网页，用 Codex 内置浏览器就够了。
- 但如果要进入你已经登录的网站，比如 Gemini、B站、YouTube、Gmail、Notion，以及各种 SaaS 工具后台——这些地方靠的是一个现成的登录状态，那就得用 Chrome 插件了。

说白了，Chrome 插件的价值不是要替代内置浏览器，而是让 Codex 能真正进到你日常工作的浏览器环境里去干活。



# 常见问题

## **文件上传/下载失败怎么办**？

进 `chrome://extensions/`，找到 Codex 扩展，点"详细信息"，打开"允许访问文件网址"权限。这是最常见的坑——Chrome 默认不给扩展访问本地文件的权限。



## **支持哪些浏览器**？

目前只支持 Chrome 。 Edge 、 Arc 、 Brave 等其他 Chromium 内核浏览器*()*都不支持。别折腾了，换回 Chrome 用。



## **会影响我当前的浏览吗**？

不会。 Codex 用独立标签组，跟你正在看的页面不冲突。你可以一边让它干活一边正常用浏览器。



## **安全性怎么样**？

敏感操作会暂停确认。而且 Codex 只在后台标签组操作，不会随便动你正在用的标签。但建议——涉及支付、银行、敏感后台的操作，第一次用的时候在旁边看着它跑完，确认流程没问题再放手。
