# 第2节：入口全景地图 — 5 个入口怎么选

> 时长：约15分钟 | 类型：展示型

---

## 开场白（30秒）

上节课我们了解了 Codex 是什么。这节课我们来看一个很实际的问题：Codex 有 5 个使用入口，你应该选哪个？

别急，我帮你逐一分析，最后给你一个清晰的选择指南。

---

## Slide 2：五大入口一览（2分钟）

Codex 目前提供了五个入口，我按照使用频率给大家排个序。

第一个是桌面 App，这是最主流的入口，功能最全面。第二个是 VS Code IDE 插件，开发者首选。第三个是 Cloud 网页版，不用安装。第四个是 CLI 命令行，最灵活。第五个是在 ChatGPT 里使用 Codex。

每个入口有不同的适用场景，我们逐一来看。

---

## Slide 3-7：逐一讲解（8分钟）

先看桌面 App。下载安装后就能用，图形界面，支持 Skills、MCP、Automation 等全部功能。如果你是非技术用户或者想深度使用，这是你的主力入口。

再看 VS Code IDE。安装插件后，你可以在编辑器里直接调用 Codex。选中一段代码，让 AI 帮你修改或重构。如果你是开发者，这个入口会非常顺手。

Cloud 网页版的好处是不用安装任何东西，打开浏览器就能用。适合换电脑或者临时需要用的场景。不过功能相对基础。

CLI 命令行是开发者最爱的入口。在终端里输入一行命令就能让 Codex 干活。它还支持远程模式，可以连接云服务器来修 Bug。

最后是 ChatGPT 中的 Codex。如果你已经有 ChatGPT Plus 订阅，可以直接在对话里启用 Codex 能力。最易上手，但功能相对有限。

---

## Slide 8：功能对比表（2分钟）

这张对比表可以帮你快速判断。

从安装、功能覆盖、适合人群三个维度对比，桌面 App 和 VS Code 是功能最全的两个入口，推荐度五颗星。

Cloud 和 ChatGPT 适合轻度使用，推荐度三颗星。CLI 适合开发者和自动化场景，推荐度四颗星。

---

## Slide 9：按角色选入口（2分钟）

最后给你一个简单的选择指南。

如果你是非技术用户，就用桌面 App 就够了。如果你是开发者，推荐 VS Code 加 CLI 的组合。如果你只是偶尔用用，Cloud 或 ChatGPT 入口最方便。

我的建议是：先用桌面 App 入门，熟悉之后再按需扩展其他入口。

---

## 总结（30秒）

好，这节课的要点。Codex 有 5 大入口，各有侧重。桌面 App 功能最全，是日常主力。开发者推荐 VS Code 加 CLI。按角色和需求灵活选择。
