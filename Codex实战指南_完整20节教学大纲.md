# 模块一：认知篇 —— 为什么 Codex 是你的 AI 提效首选

---

## 第01节：认识 Codex：为什么它比 ChatGPT 更适合做你的 AI 助手（12 min | 展示）

**教学目标**：建立"Codex是任务代理而非聊天工具"的正确认知框架，能独立判断不同AI工具的适用场景。

**内容结构**：

1. **开场：为什么需要重新认识AI助手（1.5 min）**
   - 核心知识点：当前AI工具泛滥，但真正能"替你干活"的不多；引出核心问题——ChatGPT、Copilot、Codex到底有什么区别？
   - 教学方法：讲解+屏幕展示三个工具的logo和官网首页

2. **深度对比：ChatGPT vs Copilot vs Codex 能力范式对照表（4 min）**
   - 核心知识点：
     - ChatGPT：对话式问答，一问一答，适合"我知道我要问什么"
     - Copilot：编码辅助补全，适合"我在写代码时需要灵感"
     - Codex：代理式任务执行，读文件→改代码→跑测试→迭代验证的自主闭环，适合"我有一个任务，你帮我做完"
     - 用同一任务做对照演示：要求三个工具分别"写一个Python脚本读取CSV并生成柱状图"
   - 教学方法：录屏对比演示+讲解
   - 旧版补充（C-01）：能力范式对照表，核心论据"不是更好，是不同范式"

3. **Codex的核心能力拆解（3 min）**
   - 核心知识点：读取项目代码库、自主规划执行步骤、运行测试验证结果、迭代修复直到完成
   - 教学方法：录屏展示Codex处理真实小任务的完整流程

4. **"任务特征→入口选择"决策框架（2 min）**
   - 核心知识点：临时问答→ChatGPT；本地项目迭代→桌面App；团队代码开发→VS Code；远程部署→CLI/Cloud
   - 教学方法：讲解+屏幕展示决策流程图
   - 旧版补充（C-02）：防止学员"拿着锤子找钉子"

5. **诚实预期管理（1.5 min）**
   - 核心知识点：不能替你做决策、不能替代业务理解、需要人工验证、复杂架构仍需人类把关
   - 教学方法：讲解+屏幕展示"诚实预期管理清单"
   - 旧版补充（A-01）：降低退课率和差评率

**核心产出**：建立正确认知框架 + 1张"任务-入口"决策速查卡

**配套资料**：能力范式对照表PDF、决策框架图、诚实预期管理清单

**课后任务**：列举你日常工作中3个最频繁的任务，判断它们分别适合用哪个工具处理。

**录制要点**：三个工具对比时务必用同一任务做对照；预期管理语气要诚恳积极。

## 第02节：Agent 新范式：从"提问"到"委托"的思维转变（15 min | 展示）

**教学目标**：掌握从"向AI提问"到"向Agent委托任务"的思维转变，能使用AGENTS.md为Codex建立"信任框架"。

**内容结构**：

1. **认知破冰：从"聊天机器人"到"自主代理"（3 min）**
   - 核心知识点：旧版金句"Codex不是聊天机器人，是能读文件、改代码、跑测试的代理"；ChatGPT像"咨询顾问"，Codex像"外包程序员"
   - 教学方法：讲解+类比图示+录屏展示Codex自主执行精彩片段
   - 旧版补充（B-01/C-01）：认知破冰的最高效武器

2. **"提问" vs "委托"：两种思维模式深度对比（4 min）**
   - 核心知识点：提问思维（一问一答，自己复制粘贴）vs 委托思维（给目标+上下文+交付要求，Codex自主完成）
   - 教学方法：讲解+录屏对比演示同一需求用两种方式处理的效率差异

3. **为什么需要"立规矩"：AGENTS.md底层机制解释（4 min）**
   - 核心知识点：每个新对话上下文都是空白的，Codex会"失忆"；AGENTS.md是给Codex的"项目说明书"；五大组成部分：项目概览/常用命令/代码规范/安全边界/交付要求；写作原则：越具体越好
   - 教学方法：讲解+录屏展示AGENTS.md文件位置和写法+好坏对比案例
   - 旧版补充（B-08/C-13）：从底层机制解释"思维转变"的必要性

4. **委托思维实战：3个真实场景演练（3 min）**
   - 核心知识点：
     - 场景1（职场）：委托Codex整理会议纪要并生成待办清单
     - 场景2（自媒体）：委托Codex将一篇文章改写成3个不同平台版本
     - 场景3（副业）：委托Codex分析竞品数据并输出对比表格
   - 教学方法：讲解+屏幕展示3个场景的标准委托话术卡

5. **模块小结（1 min）**
   - 核心知识点：把每次打开Codex时的第一句话，从"请问..."改成"请帮我..."

**核心产出**：掌握Agent委托思维 + 1份AGENTS.md最小可用模板

**配套资料**："提问vs委托"思维对比图、AGENTS.md最小可用模板、3个真实场景委托话术卡

**课后任务**：选择你本周的一个真实工作任务，用"委托思维"重写任务描述（不少于100字，包含：目标、上下文、交付要求）。

**录制要点**：AGENTS.md部分一定要展示"好vs不好"的对比案例；类比用"实习生"比技术术语更易懂。

## 第03节：你的第一个 Codex 项目：5分钟完成一个真实任务（18 min | 录屏）

**教学目标**：独立完成首个Codex自动化任务，掌握项目模式的选择逻辑和7步法标准流程。

**内容结构**：

1. **课前准备：下载安装与界面初识（1 min）**
   - 核心知识点：确认已安装Codex桌面App；认识主界面布局
   - 教学方法：快速录屏展示主界面，关键按钮高亮标注

2. **项目模式 vs 对话模式：选择逻辑（2 min）**
   - 核心知识点：对话模式（一次性交互）vs 项目模式（持久上下文+工作目录+文件访问）；本节课必须用项目模式
   - 教学方法：录屏实操演示两种模式的切换
   - 旧版补充（A-03/B-04）：防止学员后续误用对话模式

3. **实操演示：7步法完成第一个Codex项目（10 min）**
   - 3.1 **步骤1-2：创建文件夹与选择模式（1.5 min）**
     - 核心知识点：新建文件夹→打开Codex→New Project→Local Project→添加工作目录
     - 教学方法：录屏实操，每点击一个按钮后停顿1秒
     - 旧版补充（C-04）：7步法步骤1-2
   - 3.2 **步骤3：输入任务描述（2 min）**
     - 核心知识点：使用"委托思维"结构——目标+上下文+交付要求；本节课任务：创建一个每日待办清单的HTML页面
     - 教学方法：录屏实操+展示如何写好任务描述
     - 旧版补充（C-04）：7步法步骤3
   - 3.3 **步骤4-5：观察执行与预览结果（3 min）**
     - 核心知识点：观察执行日志；关键操作节点：Codex询问"是否允许执行"时的判断；预览生成的index.html
     - 教学方法：录屏实操+讲解关键节点
     - 旧版补充（C-04）：7步法步骤4-5
   - 3.4 **步骤6-7：迭代修改与完成确认（3.5 min）**
     - 核心知识点：提出修改需求→观察迭代→确认完成；重点展示"迭代"而非一次性完美输出
     - 教学方法：录屏实操
     - 旧版补充（C-04）：7步法步骤6-7

4. **上下文窗口管理技巧（2 min）**
   - 核心知识点：Codex"失忆"现象识别；解决方法：新建对话，AGENTS.md配置自动继承
   - 教学方法：讲解+录屏演示"新建对话"操作
   - 旧版补充（B-04）：解决核心痛点，大幅降低首次实操挫败感

5. **课后任务布置与资源导航（3 min）**
   - 核心知识点：三选一课后任务（文件整理脚本/个人主页/数据看板）；Prompt模板卡介绍
   - 教学方法：讲解+屏幕展示模板卡+下节课预告
   - 旧版补充（A/B配套）：Prompt模板卡5-15张

**核心产出**：完成首个自动化任务（待办清单网页）+ 1份"首次任务检查清单"（7步法）

**配套资料**：7步法检查清单、Prompt模板卡（任务描述/迭代修改/安全确认）、本节课完整代码文件包

**课后任务**：独立使用Codex完成一个个人小项目，截图提交到课程社群。

**录制要点**：全程录屏，鼠标移动缓慢清晰；迭代修改分两次小改，展示"渐进式优化"。

## 第04节：桌面 App 深度入门：界面、工作区与核心操作（15 min | 录屏）

**教学目标**：学员能够独立在桌面App中完成从新建项目到执行任务的完整操作闭环。

**内容结构**：

1. **界面导航与核心区域速览（3 min）**
   - 核心知识点：五大区域——对话区、文件树、工具栏、状态栏、设置入口；对话框四要素（添加上下文/切换模型/控制权限/选工作目录）
   - 教学方法：录屏+鼠标高亮跟随
   - 旧版补充（B-03/C-04）：对话框四要素作为记忆锚点

2. **Chat 模式 vs Project 模式：选对才能做对（4 min）**
   - 核心知识点：Chat（一次性交互）vs Project（持久上下文+工作目录）；决策口诀："改文件→Project；问问题→Chat"
   - 教学方法：录屏分屏对比
   - 旧版补充（B-03/C-04）：完整选择逻辑

3. **Project 模式下的标准工作流：7步法实操（5 min）**
   - 核心知识点：Step1创建文件夹→Step2选择模式→Step3添加目录→Step4输入任务→Step5观察执行→Step6预览结果→Step7迭代修改
   - 教学方法：完整录屏演示真实任务
   - 旧版补充（C-04）：7步法作为核心骨架

4. **设置面板分层速览：新手只调这两项（2 min）**
   - 核心知识点：常规设置（主题/语言/模型）+ 权限设置（Suggest/Auto Edit/Full Auto默认）
   - 教学方法：录屏快速翻页，重点停留常规+权限
   - 旧版补充（C-04）：设置面板分层速览

5. **移动端补充：手机端不是远程桌面（1 min）**
   - 核心知识点：手机端定位是"跟进+审批"工具；适合查看进度/回复澄清/审批命令；不适合大范围review/高风险操作
   - 教学方法：讲解+手机界面截图
   - 旧版补充（B-05/C-05）：手机端定位说明

**核心产出**：学员能独立操作Codex桌面App完成全流程

**配套资料**：7步法流程图PDF、Chat vs Project决策速查卡、对话框四要素记忆卡

**课后任务**：在本地创建一个项目文件夹，用Project模式完成一次真实任务。

**录制要点**：录屏必须开启鼠标高亮和按键显示；7步法演示时每步口头报出"第X步"。

## 第05节：权限体系与执行模式：让 Codex 安全地帮你做事（15 min | 录屏）

**教学目标**：学员能够根据任务特征正确选择执行模式，并建立个人安全使用规范。

**内容结构**：

1. **为什么权限是第一课：一个真实踩坑故事（2 min）**
   - 核心知识点：展示"Full Auto模式下Codex误删文件"的模拟场景
   - 教学方法：录屏演示+讲解

2. **三档执行模式详解与选择口诀（4 min）**
   - 核心知识点：Suggest（只给建议）/ Auto Edit（自动编辑需确认）/ Full Auto（全自动）；口诀："新项目先Suggest，批量改用Auto Edit，边界清晰才Full Auto"
   - 教学方法：录屏切换三种模式执行同一任务
   - 旧版补充（B-07/B-10）：权限三档选择口诀

3. **风险五层模型：你的数据安全地图（4 min）**
   - 核心知识点：第一层文件系统→第二层网络→第三层命令→第四层凭据→第五层数据
   - 教学方法：讲解+可视化五层金字塔图+逐层录屏演示
   - 旧版补充（B-10/C-14）：风险五层模型

4. **低风险 vs 高风险任务清单（3 min）**
   - 核心知识点：低风险（批量重命名/生成文档/格式化代码）vs 高风险（删除文件/执行部署/修改配置/访问生产数据库）
   - 教学方法：讲解+屏幕展示对照表
   - 旧版补充（B-07/C-12）：任务清单

5. **安全提示词模板：给 Codex 戴上紧箍咒（2 min）**
   - 核心知识点：安全约束Prompt模板、范围限定Prompt模板、回滚提示Prompt模板
   - 教学方法：讲解+直接展示可复制的Prompt文本
   - 旧版补充（B-10/C-12）：安全提示词模板

**核心产出**：个人安全使用规范 + "任务→模式→防护"三步决策法

**配套资料**：权限三档口诀卡、风险五层安全地图、任务对照表、安全提示词模板库

**课后任务**：为自己的一个真实项目填写"权限使用规范表"。

**录制要点**：演示"误删风险"时必须使用测试目录；五层模型每层都要对应真实操作场景。

## 第06节：Skills 系统：扩展能力边界的万能插件（18 min | 录屏）

4. **适合写成 Skill 的场景清单（3 min）**
